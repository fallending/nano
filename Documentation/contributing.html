Contributing Code

If you have any suggestions for Nano development, or would like to submit bugs or patches, then please contact me.

Submitted code will be placed under the Nano licence, and you should review the terms of that document before contributing code.

Custom Views

Future releases of Nano will extend the number of custom views in the library, in order to provide views that are currently Cocoa-specific or restricted to applications from Apple.

Examples of these views could be:

NCoreImageView Contains a CoreImage view
NCoreVideoView Contains a CoreVideo view
NLabelView Similar to the Finder label color selector
NMembraneButton Similar to the Finder Find window
NHoverText Similar to the Safari bookmark control
NProgressPie Similar to the Xcode pie-chart progress indicator
NStatusBar Similar to the Xcode status bar
NDialView Similar to the Cocoa circular slider control
NGlassButton Similar to the Cocoa "round textured button" control
NLevelIndicator Similar to the Cocoa "level indicator" control
NMetalButton Similar to the Cocoa "square textured button" control
NSplitterView A "splitter" view that splits two views
NBezelView A "bezel" view that draws a translucent bezel

If you are interested in developing one of these views for Nano, please contact me first to ensure that nobody else is working on a similar class.

Application Features

The core application framework in Nano will also be extended in future releases, to provide additional features that are becoming standard in many modern applications.

Some examples of these features could be:

Recent Menu A File>Recent menu to track documents
Software Update A Software Update system that checks online for updates
Undo Manager An undo/redo system for tracking state changes

If you are interested in developing one of these, or have additional features that you feel should be a standard part of a modern application framework, please let me know.
